The km node marginal is the effect an extra 1MW of generation at that node has on total MW.km. It is the combined effect of incrementally increasing and or decreasing planned transfer flows on all the pathways from the node to the reference node. A -ve marginal km will result from extra generation in a largely importing area or extra demand in a largely exporting area. It will be zero at the reference node.

Branches connect nodes together to form a power network and are used to model overhead power lines, undeground cables, transformers and interconnectors etc.

This is the unique identifier for the the branch. When creating a new branch this field can be left blank and the program will generate a unquie value for this field automatically.

Branches connect two nodes that are labelled Node1 and Node2. More than one branch can be used to connect the same nodes. Power flow will be positive if the flow is from Node1 to Node2 and negative if the flow from Node2 to Node1.

Node1 => Node2 (positive powerflow)

Node2 => Node1 (negative powerflow)

This is the type of branch.

When the two nodes are of the same voltage but in different locations then the following types of Branch are valid:-

  • OHL (overhead line)
  • Cable
  • Composite (mixture of overhead line and cable)
  • HVDC (High Voltage DC line)

For the same voltages at the same location these types are available:-

  • QB (Quad Booster)
  • Series Capacitor

For different voltages at the same location then this type is valid:-

  • Transformer

The length of the overhead line (only used when type is OHL or Composite)

Length of cable (only used when type is Cable or Composite)

Reactance of the branch.

This is in units of of % on 100MVA base and means there is 1% of line voltage drop for a flow of 100MW. As reactive volt drops are 90 degrees ahead of current it manifests as a 0.01 radian voltage phase shift from send end to receive end per 100MW of flow.

Resistance of the branch with units of % on a 100MVA base.

The power capacity of the branch in MW.

Whether this branch is tripped or not. A tripped branch will not allow any power flow between its nodes.

The amount of free power avaiable for this branch in MW and is simply Capacity - Flow.

The power flow along the branch in MW.

The power flow along the branch given as a percentage of the branch capacity.

The effective length (in km) reflecting how cables and lower voltage lines are deemed longer in proportion to the extra cost of using them to establish a MW.km of capability.

The branch MW.km is the product of the branch power flow and its deemed effective length (in km) reflecting how cables and lower voltage lines are deemed longer in proportion to the extra cost of using them to establish a MW.km of capability.

The sum of all branch MW.km gives the total network size needed to carry the planned transfer flow pattern (NB without extra parallel path capacity for security).

The branch loss is an estimate of power loss (in MW) resulting from heating effects caused by power flowing through conductor resistance R (which has units of % 100MVA base but gives a volt drop in phase with line voltage).

R is typically less than X/20 so voltage effects are neglected in a linearised load flow although the MW loss can be summed up.

Total system loss is usually 1.5% to 2% of total demand.

Controls are model artifacts that can control power flow through the network and whos values can be optimised to achieve certain goals such as maximising power flow across boundaries.

This is a unique identifier for the control. When the control is associated with a branch then the code will be the same as the branch.

This is either a Node or Zone associated with the control. For branch controls these will be the nodes used by the branch but for other types of controls they can be a Node or a zone dependent on the type

This is the type of control. Types can be divided into two broad categories: those associated with a branches and others.

Controls associated with branches control the power flow along their associated branch and can be one of these types:-

  • QB - These model Quad Boosters
  • Series Capacitor - These model the effects of series capacitors
  • HVDC - These model HVDC links

Other types of controls are:-

  • DecInc - These directly control the demand of the two nodes they are associated with
  • InterTrip - These directly control the demand of a single node and a whole zone
  • Transfer - These control demand at the two zones they are associated with

Boundaries typically divide the network in two and are useful to show general flows around the network.

A Boundary is defined by assigning it a collection of Zones. Then all nodes that belong to the zones in the boundary are assumed to be one side of the boundary with all remaining nodes being the other side of the boundary.

The unique identifier for the boundary

The collection of geographical zones defining the boundary

A Zone is a specific geographical area containing a set of nodes. A node must belong to one and only one zone.

unique identifier for the zone

Generator models are used to specify the energy mix used to satisfy the model's demand for electricity.

Each model defines a percentage contribution from each energy source used by the generators in the model.

Type of energy source. These correspond to the different types of generators defined.

The total Maximum Entry Capacity for the given energy source.

This is the total Max TEC for each generator that uses this energy source.

If set, the contribution for this energy source will be auto calculated.

At least one energy will need to be auto calculated to ensure that demand = generation.

The percentage contribution for this energy source

When a Boundary is not specified BoundCalc will perform so called Planned Transfer analysis. The user can select a different Generation model (that changes the fuel mix used to satify demand), trip one or branches and then see the effect on the power flow in the network.

When a Boundary is selected BoundCalc will perform so called Boundary Analysis.

This will calculate the maximum flow that can be achieved across the boundary until one or more branches of the network reach their maximum capacity.

Multiple runs using different trip configurations are performed. An Intact trip, that has no trips defined, and then runs for each combination of single and double trips of the branches forming the boundary.

The results are listed in a new tab called All trip results.

Any user-defined trips will be ignored and the boundary branches will be shown in green on the map.

Selecting a specific boundary trip will show the results for the given boundary trip scenario.

These are the number user selected trips when using BoundCalc in Plannned Transfer mode.

The selected Generation Model to use when performing a flow calculation.

A Generaion Model defines the fuel mix to use when satisfying demand.

Using manual set point mode allows a user to finely tune the control's set points to see the effect they have on the network.

To use manual mode, the user will need to first perform a run using automatic mode (to obtain initial values) after which manual mode can be selected. When put into manual mode the user can modify the control set points by editing the values in the Set Point> column of the Controls data table.

When clicked, this will cause so called Node Marginals to be calculated. Node Marginals are the data in Node columns TLF and km.
